## Test Case: Register
Validation / Error case
1. ส่ง {} (ว่าง ไม่มี field อื่น ๆ)
2. ส่ง username ยาวเกิน 30 ตัวอักษร
3. ส่ง email ไม่ตรงตาม format เช่น "test@wrong"
4. ส่ง password สั้นเกิน 6 ตัวอักษร
5. ส่ง field อื่น ๆ ที่ไม่ได้อยู่ใน DTO (เช่น role, isAdmin)
6. ส่งซ้ำ email ที่เคย register แล้ว → expect duplicate error
7. ส่งซ้ำ username ที่เคย register แล้ว → expect duplicate error

Success case
8. ส่งข้อมูลถูกต้องครบถ้วน → expect 201 Created + message success
9. ส่งข้อมูลที่ optional field (ถ้ามี เช่น bio หรือ avatar) → ควร save ได้ถูกต้อง

## Test Case: Login
Validation / Error case
1. ส่ง {} (ว่าง ไม่มี field)
2. ส่ง email ไม่ตรง format เช่น "wrong-email"
3. ส่ง password สั้นเกิน 6
4. ส่ง email ที่ไม่มีอยู่ในระบบ → expect “user not found”
5. ส่ง email ถูกต้องแต่ password ไม่ตรง → expect “invalid credential”

Success case
6. ส่ง email+password ถูกต้อง → expect 200 OK พร้อม accessToken / refreshToken
7. Login ได้แล้ว → token ที่ได้สามารถนำไปเรียก endpoint ที่ require auth ได้

## Test Case: Get Profile
Validation / Error case
1. ไม่ส่ง token → expect 401 Unauthorized
2. ส่ง token ที่หมดอายุแล้ว → expect 401 Unauthorized
3. ส่ง token ไม่ถูกต้อง (random string) → expect 401 Unauthorized

Success case
4. ส่ง token ถูกต้อง → return ข้อมูล profile ของ user เอง
5. ส่ง token ถูกต้องของ user A → ต้องไม่สามารถเห็น profile user B (ถ้ามี route /profile/:id)
6. Response format ตรงตามที่ระบบ define (ApiResponse / ApiResponseFormat)
